# AWS Day 02 – Identity and Access Management (IAM)

## Objective

The objective of this lab was to understand AWS Identity and Access Management (IAM), create users, groups, policies, and roles, and learn AWS security best practices.

---

## Topics Covered

- Introduction to IAM
- Root User
- IAM User
- IAM Group
- IAM Policy
- Managed Policies
- Customer Managed Policies
- Inline Policies
- IAM Roles
- Authentication
- Authorization
- Principle of Least Privilege
- Multi-Factor Authentication (MFA)

---

## Practical Activities Performed

- Explored IAM Dashboard
- Created IAM User (devopsuser)
- Created Developers Group
- Attached AmazonS3ReadOnlyAccess Policy
- Added User to Group
- Created Custom EC2 ReadOnly Policy
- Explored IAM Roles
- Verified Root Account MFA
- Learned AWS Security Best Practices

---

## Key Learnings

- IAM controls access to AWS resources.
- Root User should never be used for daily tasks.
- Groups simplify permission management.
- Policies define permissions using JSON.
- Roles provide temporary credentials to AWS services.
- MFA improves account security.
- Principle of Least Privilege reduces security risks.

---

## Skills Gained

- IAM User Management
- Group Management
- Policy Creation
- Security Best Practices
- AWS Console Navigation
- Understanding JSON Policies

---

## Conclusion

IAM is the foundation of AWS security. Understanding users, groups, policies, and roles is essential before working with other AWS services.
