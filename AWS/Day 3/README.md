# AWS Day 03 – Amazon EC2 (Elastic Compute Cloud)

## Objective

The objective of this lab was to understand Amazon EC2, launch a virtual server, connect using SSH, install Apache, deploy a web page, and learn the EC2 lifecycle.

---

## Topics Covered

- Introduction to Amazon EC2
- Amazon Machine Image (AMI)
- Instance Types
- Key Pairs
- Security Groups
- Elastic Block Store (EBS)
- User Data
- Elastic IP
- EC2 Lifecycle
- SSH Connectivity
- Apache Web Server

---

## Practical Activities Performed

- Launched an Ubuntu EC2 instance
- Configured Security Groups
- Connected to EC2 using SSH
- Updated the Ubuntu server
- Installed Apache Web Server
- Started and enabled Apache
- Modified the default Apache web page
- Hosted a custom HTML page
- Explored the EC2 lifecycle (Start, Stop, Reboot)
- Learned about Elastic IP and User Data

---

## Key Learnings

- EC2 provides scalable virtual servers in the cloud.
- AMIs are templates used to launch EC2 instances.
- Security Groups act as virtual firewalls.
- Key Pairs provide secure SSH access.
- EBS volumes provide persistent storage.
- User Data automates instance initialization.
- Elastic IP provides a static public IP address.

---

## Skills Gained

- EC2 Instance Management
- SSH Connectivity
- Apache Installation
- Linux Administration on EC2
- Security Group Configuration
- Basic Web Hosting on AWS

---

## Conclusion

Amazon EC2 is one of the core AWS services. Learning how to launch, configure, secure, and manage EC2 instances is essential for every AWS and DevOps engineer.
