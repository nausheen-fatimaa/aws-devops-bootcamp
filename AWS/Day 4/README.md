# AWS Day 04 – Amazon S3 (Simple Storage Service)

## Objective

The objective of this lab was to understand Amazon S3, create buckets, upload objects, enable versioning, configure lifecycle rules, and host a static website.

---

## Topics Covered

- Introduction to Amazon S3
- Buckets and Objects
- Storage Classes
- Bucket Policies
- IAM Policies vs Bucket Policies
- Versioning
- Lifecycle Rules
- Static Website Hosting
- Public Access Configuration

---

## Practical Activities Performed

- Created an S3 Bucket
- Uploaded HTML files and other objects
- Enabled Bucket Versioning
- Created a Lifecycle Rule
- Enabled Static Website Hosting
- Configured Bucket Policy for public read access
- Hosted a Static Website
- Accessed the website using the S3 endpoint

---

## Key Learnings

- Amazon S3 is an object storage service.
- Buckets store objects such as images, videos, and documents.
- Versioning protects against accidental deletion and overwrites.
- Lifecycle Rules reduce storage costs automatically.
- Static websites can be hosted directly from S3.
- Bucket Policies control access at the bucket level.

---

## Skills Gained

- S3 Bucket Management
- Object Uploading
- Website Hosting
- Bucket Policy Configuration
- Storage Management
- AWS Console Navigation

---

## Conclusion

Amazon S3 is a highly durable, scalable, and cost-effective storage service widely used for backups, media storage, application assets, and static website hosting.
