# Key Learnings

- Git tracks source code changes.
- Git allows multiple developers to work simultaneously.
- Branching helps develop new features independently.
- Commits create checkpoints.
- GitHub stores repositories in the cloud.

---

# Real World DevOps Usage

In DevOps, Git is used to:

- Store Infrastructure as Code (Terraform)
- Store Dockerfiles
- Store Kubernetes YAML files
- Maintain Jenkins Pipelines
- Collaborate with teams
- Track application changes
- Rollback to previous versions
